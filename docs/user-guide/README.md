# user-guide

Reference pages for the final BigBrotr runtime and architecture contracts.

## What Lives Here

- `index.md`: landing page for the reference section.
- `architecture.md`: DAG layering, runtime shape, and system contracts.
- `services.md`: service responsibilities and boundaries.
- `database.md`: shared schema and derived data model.
- `read-side.md`: read-core, public resources, and bounded query surface.
- `deployments.md`: deployment folder contract and storage-profile model.
- `configuration.md`: YAML configuration reference.
- `monitoring.md`: metrics, alerting, dashboards, and logs.

## Rules

- Keep this section reference-oriented rather than tutorial-shaped.
- Reflect runtime contracts from code and deployments, not aspirational
  design notes.
